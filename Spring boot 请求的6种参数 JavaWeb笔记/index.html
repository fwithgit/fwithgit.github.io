Spring boot 请求的6种参数 JavaWeb笔记

发表于2024-03-09|更新于2024-03-10

参数是什么

  大概就是网址后面加的一大堆东西，然后服务器会根据你的请求参数做出响应，参数有很多种，下面就是在介绍各种类型的参数。

测试工具-Postman

  在接下来的请求参数例子中如果使用浏览器来进行测试会比较麻烦，Postman是一款可以模拟浏览器发送请求的工具。
  下载完程序双击直接安装即可，在首页完成登录和个人工作区的创建，进到具体界面，可以选择GET、POST等请求，输入地址栏等。非常容易上手。
Postman官方下载地址：Download Postman | Get Started for Free

简单参数

  简单参数的发送选择get、post均可，如果是get请求在地址栏以“?请求参数&其他请求参数”的格式添加，或者直接在Postman下方出现的表格里添加，点击表格左边的勾可取消参数的使用。post请求忘了…大差不差的。

接收参数的方法

  发送了参数后服务端当然不会自己搞定，还需要你自己来弄。

原始方式

  使用Servlet接收，较为麻烦，且黑马程序员那里没有详细的讲解这部分内容…

Spring Boot方式

  先使用注解@RequestMapping("/路径id")来使得在网址输入路径id能找到相应的函数，然后创建一个和路径id一样的函数，其形参和请求参数名要保持一致即可。

  如果形参和请求参数不一致，可以使用@RequestParam这个注解来使他们等价，例如服务端的形参是username，但客户端的请求参数是name，可以写成@RequestParam(name = "name")String username, integer age，这样改了以后，username会变为一个必须的参数，客户端不输入这个参数会导致错误，要改的话要在等价后面加上required = false，如public String simpleParam(@RequestParam(name = "name", required = false) String username, Integer age)

  Int和Integer的区别

实体参数

  首先新建好实体的类，简单的实体类可以新建一个pojo的包存放，如果这个类里面还包含了别的类，客户端发送参数时要类A.属性a这样赋值

数组集合参数

数组

  数组和简单参数差不多，客户端直接把值一个个传过来就好了

集合

  集合和数组的区别主要区别其实只有一个，就是要用@RequestParam来绑定集合作为参数。数组输出的时候是使用Arrays.toString(数组)来输出，集合直接输出变量即可。客户端的请求是一样的。

日期参数

  日期参数需要使用注解DateTimeFormat设置日期格式，然后客户端发送时需要补齐0否则会出错，另外月份的简写和分钟的简写是一样的，都是mm，但是可以通过大小写来避免冲突。

Json参数

  发送json参数测试时，Postman选择Post请求，还要选择请求body里面的raw，再选择json。

路径参数

  路径参数和其他参数不一样，其他参数请求时要加上函数值，路径参数直接输入路径即可
